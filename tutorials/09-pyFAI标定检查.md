# pyFAI 标定检查 — 使用教程口播稿

---

## 开场

大家好，这一集我们来讲解 pyFAI 标定检查功能。

pyFAI_calib2 是 pyFAI 自带的一个图形化标定工具，用于确定探测器的几何参数。X-FAIS 的标定检查功能不是用来做标定的，而是帮你检查系统中 pyFAI_calib2 是否正确安装、能否正常启动。

---

## 页面布局

页面比较简单：

- 顶部：检查按钮和状态摘要
- 下方：详细状态卡片

---

## 第一步：检查状态

点击"检查状态"按钮，软件会检测系统中 pyFAI_calib2 的安装情况。

检测完成后，页面会显示一个状态摘要：
- 绿色标签：一切正常，可以使用
- 黄色标签：部分可用，有警告
- 红色标签：不可用，需要修复

---

## 第二步：查看详细信息

状态卡片分为两个部分：

### 嵌入式 Python 环境
这是 X-FAIS 内嵌的 Python 环境，显示：
- **Python 版本**：内嵌的 Python 版本号
- **calib2 路径**：pyFAI_calib2 可执行文件的路径
- **是否可用**：是否检测到 pyFAI_calib2
- **Qt 绑定**：pyFAI_calib2 需要 Qt 图形界面库，检查是否有可用的 Qt 绑定

### 系统 Python 环境
这是你电脑上已安装的 Python 环境，显示同样的信息。

如果两个环境都有可用的 pyFAI_calib2，你可以选择使用哪个。

---

## 第三步：启动标定

如果检测到可用的 pyFAI_calib2，页面会显示启动按钮。点击后会启动外部的 pyFAI_calib2 图形界面。

注意：pyFAI_calib2 是一个独立的外部程序，启动后会在 X-FAIS 之外打开一个新窗口。

---

## 为什么需要这个功能

pyFAI_calib2 的安装有时候会遇到问题，特别是 Qt 绑定。常见的问题包括：

1. **Qt 绑定缺失**：pyFAI_calib2 需要 PyQt5 或 PySide2，但 Python 环境中没有安装
2. **版本不兼容**：pyFAI 版本和 Qt 版本不匹配
3. **路径问题**：pyFAI_calib2 安装了但不在系统 PATH 中

标定检查功能可以快速诊断这些问题，告诉你具体是哪个环节出了问题。

---

## 实际应用场景

1. **首次使用前**：确认 pyFAI_calib2 可以正常使用
2. **环境变更后**：Python 环境更新后检查兼容性
3. **问题排查**：标定功能不工作时，先用这个检查工具诊断
4. **选择环境**：在嵌入式 Python 和系统 Python 之间选择更合适的环境

---

## 注意事项

1. 这个功能只是检查和启动，不会修改任何配置
2. pyFAI_calib2 是外部程序，它的使用方法不在本教程范围内
3. 如果检查结果显示不可用，可能需要重新安装 pyFAI 或安装 Qt 绑定
4. 嵌入式 Python 环境的 pyFAI_calib2 可能缺少 Qt 绑定，这是正常情况
5. 系统 Python 环境如果有完整的 pyFAI 安装，通常可以直接使用
